01 — How we built it

OUR APPROACH

Two inference paths, one tactical UI. A custom-trained PyTorch classifier for single-image threat verdicts with a per-class confidence breakdown, plus a real-time YOLOv8 surveillance loop that watches a live camera feed and auto-snapshots threats. This page explains every layer.

System Overview

Two engines, one frontend

The site exposes two complementary detection modes. The Detection Console (/detect) accepts a still image and returns a calibrated threat / not-threat verdict with per-class probabilities, an explainable analysis card, and a downloadable PDF report. The Live Surveillance page (/live) opens a webcam, runs YOLOv8 on every other frame, and saves an annotated snapshot to disk whenever a monitored class crosses the confidence threshold.

Problem

Why marine detection is hard

Marine scenes are notoriously difficult for vision models. Sun glare, sea spray, occlusion by waves, and extreme variations in object scale all make naive classifiers brittle. Add the long tail of rare-but-critical classes (drifting debris, partly-submerged species) and you have a problem that demands careful preprocessing, a robust backbone, and aggressive evaluation.

02 — Classifier Architecture

The model

The custom model is a CNN classifier (ResNet-style) fine-tuned on a curated marine dataset and saved as a full nn.Module via torch.save. It produces a binary verdict — threat vs not threat. Inference runs on CPU by default and on CUDA when available. Inputs are 224×224 RGB, normalized with the dataset's own mean and standard deviation.

04 — Post-processing & Reasoning

From logits to a human-readable verdict

Raw logits go through softmax to get class probabilities. The Detection Console's threshold slider filters predictions below the chosen confidence. The frontend then derives a tier (very high / high / moderate / low) and writes a one-paragraph plain-language explanation of the verdict, all rendered into the analysis card and the downloadable PDF report.

probs = torch.nn.functional.softmax(logits, dim=1)[0] all_scores = {CLASSES[i]: float(probs[i]) for i in range(len(CLASSES))} verdict = top_label # "threat" | "not threat" tier = "very high" if conf >= 0.9 else "high" if conf >= 0.75 else "moderate" if conf >= 0.6 else "low"
05 — Training

How the model learned

Training is documented in model/Training_Methodology.ipynb. The dataset (downloaded from the team's Google Drive) holds 875 real marine images across two classes, split into training and validation folders. We use cross-entropy loss, an Adam optimizer with step-decay learning rate, and standard photo augmentations to fight overfitting.

06 — Live Surveillance (YOLOv8)

Real-time detection on a webcam

The Live page runs yolov8n.pt (Ultralytics, COCO-pretrained, ~6 MB) on every other frame from a webcam capture. Boats are flagged as threats; persons, bicycles, birds, and a few other monitored COCO classes are drawn but not alarmed. Detections are annotated directly on the frame, encoded as JPEG, and pushed over an MJPEG stream. A background thread owns the loop; the page polls /live/status once a second for telemetry (FPS, recent detections, snapshots).

07 — Deployment

Flask serves both engines

The classifier weights are loaded once at startup (weights_only=False for the full-module pickle). The YOLO model lazy-loads on first start of the live stream and is cached for the session. Every page is server-rendered Jinja2; pages are progressively enhanced with vanilla JavaScript. No build step, no bundler.
